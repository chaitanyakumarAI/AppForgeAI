 Stage 1: Intent Extractor

            
Parses raw NL into a structured IntentSpec. Lists ambiguities, makes documented assumptions, extracts entities, roles, features, and constraints. Temperature: 0.3.

          

          

            
 Stage 2: System Designer

            
Converts IntentSpec → ArchitectureSpec. Defines the full data model with typed fields and FK relations, permission matrix, API surface (CRUD + custom), and UI flow map. Temperature: 0.1.

          

          

            
 Stage 3: Schema Generator

            
Generates all 5 layers simultaneously: DB schema, API schema, UI schema, Auth schema, and Business Logic. Temperature: 0.0.

          

          

            
 Stage 4: Refinement Engine

            
The "linker" step. Receives the full schema and resolves cross-layer inconsistencies: UI→API→DB field alignment, role consistency, FK resolution. Temperature: 0.0.

          

          

            
 Validation + Repair

            
10 cross-layer checks: missing PKs, orphaned FKs, undefined roles, unmapped UI→API, mismatched response fields. On failure: targeted layer re-generation (not full retry). Max 3 repair cycles.

          

          

            
 Runtime Simulator

            
Static analysis engine that "executes" the schema: resolves all FKs, checks all routes are reachable, verifies auth gates are accessible from UI.
